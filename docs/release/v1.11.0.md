# v1.11.0 リリースノート

> v1.8.3 以降のすべての変更を含むリリースです（v1.9.0 〜 v1.11.0）。

## 主な機能

### `cdhist` / `cdj` ディレクトリジャンプ — v1.11.0

過去に訪問したディレクトリへシェル内で即復帰できるビルトインを追加しました ([#127](https://github.com/tominaga-h/jarvis-shell/issues/127))。

```bash
cdhist               # 訪問履歴を LRU 順で 1 行 1 件出力
cdhist --limit 10    # 最大件数を指定（0 で全件）
cdj                  # 全候補を fzf に渡して選択
cdj proj             # case-insensitive 部分一致で絞り込み（単一マッチなら fzf を起動せず即 cd）
```

- データソースは既存 `command_history.cwd` カラム。スキーマ追加なし、新規データ収集も不要
- 現在の cwd と存在しないパスは出力から除外
- `cdj` は単一マッチなら即 cd、複数マッチなら fzf 連携で選択
- fzf プレビューウィンドウ対応（zoxide 方式、UNIX のみ）。下 30% に `ls -Cp` で選択中ディレクトリの中身を表示。macOS は色付き、Linux は `--group-directories-first` 付き
- fzf 未インストール時は `could not find fzf, is it installed?` でユーザーに案内（exit 127）
- ESC キャンセル時は cwd 不変で exit 130

### グロブ・ブレース展開 — v1.10.0

シェル標準のファイル名展開を追加しました ([#126](https://github.com/tominaga-h/jarvis-shell/issues/126))。

```bash
ls *.toml             # グロブ
echo {a,b,c}          # ブレース展開
echo {1..5}           # 数値レンジ
echo {01..03}         # ゼロパディング保持
ls *.{txt,md}         # 組み合わせ
```

- グロブ: `*`, `?`, `[abc]`, `[a-z]`（`glob` クレート使用）
- ブレース: `{a,b,c}`, `{1..5}`, `{01..03}` ゼロパディング, `{5..1}` 降順, `{1..10..2}` ステップ, `{a..e}` 文字レンジ, ネスト, エスケープ
- 適用範囲: 外部コマンド + シェルビルトイン (`cd`, `source` 等)
- 展開順序: チルダ/環境変数 → ブレース → グロブ
- クォート尊重: `'*'`, `"{a,b}"`, `\*` は展開されずリテラル扱い
- no-match 時は zsh 互換でエラー終了（`jarvish: no matches found: <pattern>`, exit code 1）

### 起動時コマンド実行 — v1.9.0

config から指定したコマンドを起動時に実行できる仕組みを追加しました ([#49](https://github.com/tominaga-h/jarvis-shell/issues/49))。シェル起動直後に必要な初期化を自動化できます。

## バグ修正

- **Flaky テストの修正**: `exec::stderr_capture` 系テストに `#[serial]` 付与、CwdGuard と TempDir のドロップ順序を調整し、cwd 競合による flake を解消

## 内部改善

- **`src/engine/expand/` サブモジュール分割**: 旧 `expand.rs` を `alias.rs` / `basic.rs` / `brace.rs` / `glob.rs` / `pipeline.rs` / `quote.rs` の 6 ファイルに分割。CLAUDE.md「単一責務」規約に整合
- **`src/engine/builtins/wrapper/` サブディレクトリ新設**: 外部 CLI ラッパー（fzf など）をビルトイン本体と分離し、「実装」ではなく「ラッパー」であることを構造的に明示
- **dev-cycle ツールチェーン定義を CLAUDE.md に永続化**: `<!-- dev-cycle:toolchain start -->` ブロックで TEST_CMD / BUILD_CMD / LINT_CMD / CHECK_CMD / VERSION_FILE / SEMVER 設定を機械可読形式で記録
- **テストカバレッジ拡充**: グロブ/ブレース展開で約 55 件、cdhist/cdj で約 23 件のテストを追加（累計約 586 件 PASS）

## 対応プラットフォーム

- macOS (aarch64 / x86_64)
- Linux (x86_64)

## 既知の制限

- Windows は未対応（`cdj` の fzf プレビューも UNIX のみ）
- AI 機能の利用には OpenAI API キーが必要
- `cdj` の使用には `fzf` が `PATH` 上に必要
