# v1.13.0 リリースノート

> プロンプト内のコマンド置換（`$(...)` と backtick `` `...` ``）に対応したリリースです。

## 主な機能

### コマンド置換 `$(...)` / `` `...` `` に対応しました — v1.13.0

コマンドの出力を別のコマンドの引数に埋め込めるようになりました ([#266](https://github.com/tominaga-h/jarvis-shell/issues/266))。一般的なシェルと同じ `$(...)` 形式と backtick 形式の両方をサポートします。

```bash
# 基本: コマンドの出力を引数に展開
> echo $(echo hello)
hello

# backtick 形式も同じ
> echo `echo hi`
hi

# 内部にスペースを含む出力も正しく扱える
> echo $(echo a b c)
a b c

# ネスト
> echo $(echo $(echo deep))
deep

# 単語の途中への埋め込み（連結）
> echo prefix-$(echo mid)-suffix
prefix-mid-suffix
```

#### クォートと単語分割の規則

一般的なシェルと同じ規則に従います。

```bash
# クォート無し: 出力は空白で単語分割される（連続空白は畳まれる）
> echo $(printf 'a   b')
a b

# ダブルクォート内: 単語分割されず、内部の空白は保持される
> echo "$(printf 'a   b')"
a   b

# シングルクォート内: 置換は行われずリテラル
> echo '$(echo X)'
$(echo X)
```

- 置換結果の末尾の改行はすべて除去されます（先頭・中間の改行は保持）
- 置換内のコマンドが失敗（起動失敗・非ゼロ終了）した場合、外側のコマンドは中断され終了コード 1 を返します
- ネストは深さ 32 までに制限され、それを超えるとエラーになります（暴走防止）

## 内部実装

- **`src/engine/expand/command_subst.rs` 新設**: コマンド置換の中核ロジック。`$(...)` の括弧バランスによる span 検出（ネスト対応）、backtick span 検出、サブシェル実行（`run_pipeline_captured` 経由で stdout をキャプチャ）、末尾改行の除去、クォート文脈に応じた単語分割を担う。再帰は `thread_local` の深さカウンタ（RAII ガード）で上限 32 に制限
- **トークナイザ拡張** (`src/engine/expand/quote.rs`): `split_quoted` が `$(...)` / backtick span を内部空白で分割せずアトミックに取り込むよう改修。`Token` に `has_subst` / `subst_quoting` を追加し、unquoted / double-quoted の文脈を下流へ伝播
- **展開パイプライン統合** (`src/engine/expand/pipeline.rs`): 展開順序を「コマンド置換 → チルダ/環境変数 → ブレース → グロブ」とし、ダブルクォート内の置換ではグロブ／ブレースをスキップ
- **テスト追加**: トークナイザ単体・`command_subst` 単体・`dispatch` E2E を合わせて 42 件追加

## 対応プラットフォーム

- macOS (aarch64 / x86_64)
- Linux (x86_64)

## 既知の制限

- Windows は未対応
- AI 機能の利用には OpenAI API キーが必要
- コマンド置換の以下は本リリースでは未対応:
  - 直前の終了ステータス `$?` の伝播
  - backtick 内のエスケープ（`` \` ``・`\$`・`\\`）
  - 置換結果テキストへの再ブレース・再チルダ展開
  - 同一トークン内でクォート文脈が混在するケース（`"$(a)"$(b)`）の span ごとの厳密な処理。トークン単位の文脈で近似するため、内部にリテラルの `)` を含む `$(echo ")")` のような入力はパースエラーになる
  - `$(yes)` のような無限出力コマンドへのタイムアウト
