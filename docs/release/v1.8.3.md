# v1.8.3 リリースノート

> v1.6.3 以降のすべての変更を含むリリースです（v1.7.0 〜 v1.8.3）。

## 新機能

### `update` コマンド（自己更新機能）— v1.7.0

jarvish が自分自身を最新バージョンに更新できるビルトインコマンドを追加しました。

```bash
update            # GitHub Releases から最新バージョンに更新
update --check    # 新しいバージョンがあるか確認（インストールはしない）
```

- GitHub Releases から最新バイナリをダウンロードし、安全に置き換え（temp file + rename パターン）
- Homebrew インストールを自動検知し、`brew upgrade jarvish` の使用を案内
- 更新完了後に自動再起動して新バージョンを適用
- 他の jarvish プロセスへフラグファイルで更新を通知

### `update --local` オプション — v1.8.0

ソースからビルドする開発者向けに、ローカルバイナリからの更新をサポートしました。

```bash
update --local                    # デフォルトパス（target/release/jarvish）を使用
update --local /path/to/jarvish   # カスタムパスのバイナリを使用
update --check --local            # ローカルバイナリのバージョンを確認
```

## バグ修正

- **`update --check` の semver 比較バグを修正** (v1.7.2): バージョン比較ロジックが正しく動作しないケースを修正
- **SIGUSR1 自動再起動を廃止** (v1.7.2): シグナルベースの再起動をフラグファイル通知方式に変更し、安定性を向上
- **テスト実行問題を修正** (v1.8.2): `lib.rs` 作成によりテスト実行の問題を解消、`perform_local_update` のテストを安全化
- **テストの安定性向上** (v1.8.3): `sigusr1_handler_can_be_reregistered` テストのスリープ時間を調整し、CI でのフレーキーテストを解消

## テスト・品質改善

- **テストカバレッジ強化** (v1.7.3): P0/P1 項目に 27 テストを追加（累計 477 テスト）
- **`update --local` のテスト追加** (v1.8.1): Fury 監査指摘に対応し 5 件のテストを追加
- **自己更新・再起動メカニズムのテスト追加** (v1.7.1)

## 内部改善

- **Claude コマンドに frontmatter を追加**: `.claude/commands/` 配下のコマンドファイルにメタデータを追加
- **開発ルールを CLAUDE.md に追加**: 実装→完了の必須フロー（Development Cycle）を明文化
- **release コマンドの homebrew-tap 記述を変更**
- **Cargo.lock を更新**: 依存パッケージのバージョンを同期
