# v1.15.0 リリースノート

> 起動時に読み込む rc スクリプト (`rc.jsh`) を全面的に実装したリリースです。ログイン時の alias / export / 任意コマンドをファイルで管理でき、`--rcfile` / `--no-rc` CLI オプションと `source` ビルトインによるスクリプト実行にも対応しました。あわせて、補完デーモン (zsh ブリッジ / prewarm) の孤児プロセスを全終了経路で根絶する堅牢化を行っています。

## 主な機能

### rc スクリプト (`rc.jsh`) ローダー — v1.15.0

対話起動時に `~/.config/jarvish/rc.jsh` を自動で読み込み、alias・export・任意のコマンドをファイルで一元管理できるようになりました（`[startup].commands` より前に実行されます）。初回対話起動時にはコメント付きのテンプレートが自動生成されます。

```bash
# ~/.config/jarvish/rc.jsh の例
alias ll "ls -la"
export EDITOR nvim
echo "welcome back"
```

### `--rcfile` / `--no-rc` CLI オプション — v1.15.0

起動時に読み込む rc スクリプトを制御するオプションを追加しました。

```bash
jarvish --rcfile ./project.jsh   # 既定の rc.jsh の代わりに指定ファイルを読み込む
jarvish --no-rc                  # rc.jsh の読み込みを一切スキップする
```

### `source` ビルトインによるスクリプト実行 — v1.15.0

`source` ビルトインが、設定ファイル (`.toml`) のリロードに加えてスクリプトファイル (`.jsh`) の実行にも対応しました。ネストした `source` 呼び出しにも対応し、無限ループを防ぐため呼び出し深度に上限を設けています。

- FIFO・巨大ファイル・symlink など不正な入力からファイル読み書きを保護
- 実行器のセマンティクスを対話モードに揃え、途中行が失敗しても後続行は継続実行
- 深度上限を超えた場合は該当ファイル名を明示してエラー

## バグ修正

- `-c` 単体実行時、および rc.jsh 内の `exit` で終了した場合に、zsh 補完デーモンが孤児プロセスとして残る不具合を tombstone 方式で根絶
- prewarm の二重 spawn 破棄経路を有界同期 shutdown に統一し、事前ウォームアップ由来の孤児デーモンを解消

## 内部改善

- rc スクリプトローダーを実装 (`src/shell/`)
- `source` ビルトインの実行経路を対話モードと共通化し、`#[serial]` 付与により cwd 依存テストのフレークを根治
- 補完デーモンの shutdown を全終了経路で有界同期化し、kill/reap の漏れを解消
- 補完メガ機能のドキュメント総点検（`CLAUDE.md` 設定節・`reload_config` doc コメントの陳腐化を修正）

## 対応プラットフォーム

- macOS (aarch64 / x86_64)
- Linux (x86_64)

## 既知の制限

- Windows は未対応
- AI 機能の利用には OpenAI API キーが必要
- carapace 外部補完には別途 `carapace` バイナリのインストールが必要
- zsh ブリッジには `zsh` の実行環境が必要
