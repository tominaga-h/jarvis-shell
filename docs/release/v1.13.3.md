# v1.13.3 リリースノート

> 先頭トークンに相対/絶対パスを入力しても Tab 補完候補が出なかった不具合を修正したパッチリリースです。

## バグ修正

### 先頭トークンが相対/絶対パスのときファイル補完が出ない不具合を修正 — v1.13.3

コマンド位置（先頭トークン）に `./target/debug/` のような相対/絶対パスを入力しても、Tab 補完候補が一切出ず「NO RECORDS FOUND」になる不具合を修正しました。([#321](https://github.com/tominaga-h/jarvis-shell/issues/321))

**再現例:** `./target/debug/` まで打って Tab を押しても、そのパスに実行可能ファイルが存在するのに候補が表示されませんでした。

**原因:** 先頭トークンの補完は `$PATH` 内の実行可能コマンドとビルトインのみを対象にしており、相対/絶対パスがまったく補完対象になっていませんでした。

**修正:** 補完ディスパッチ (`src/cli/completer/mod.rs`) に純粋関数 `looks_like_path()` を追加しました。先頭トークンが `/` を含む、または `~` で始まる場合は `complete_path(dirs_only=false)` に委譲し、ファイル・ディレクトリの両方を補完するようにしました（`./` `../` `/abs/` `~/`、および中間に `/` を含むトークンに対応）。`complete_command`（PATH コマンド補完）は無変更のまま単一責務を維持しています。回帰テストを追加しています。

## 対応プラットフォーム

- macOS (aarch64 / x86_64)
- Linux (x86_64)

## 既知の制限

- Windows は未対応
- AI 機能の利用には OpenAI API キーが必要
