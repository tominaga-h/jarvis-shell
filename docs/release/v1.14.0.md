# v1.14.0 リリースノート

> Tab 補完システムを全面刷新したメジャー機能リリースです。`complete` ビルトインによるユーザー定義補完、carapace / zsh ブリッジ経由の外部補完、alias 対応、プロバイダ抽象化（`CompletionProvider` trait）を追加しました。([#88](https://github.com/tominaga-h/jarvis-shell/issues/88), [#89](https://github.com/tominaga-h/jarvis-shell/issues/89))

## 主な機能

### `complete` ビルトインによるユーザー定義補完 — v1.14.0

任意のコマンドに対する Tab 補完候補を、シェル上から登録・一覧・消去できる `complete` ビルトインを追加しました。

```bash
# 静的な候補を登録
> complete mytool -- start stop restart status

# 動的候補: $(...) の出力を候補として使う
> complete deploy -- $(ls environments/)

# 条件付き候補: -n で「直前の単語」に応じて候補を切り替える
> complete git -n add -- -A -p --patch

# 登録済み補完の一覧
> complete

# 特定コマンドの補完を消去
> complete -r mytool
```

- 静的候補・動的候補（`$(...)`）・条件評価（`-n <word>`）に対応
- 登録した補完は `RegistryProvider` 経由で即座に Tab 補完に反映される
- 重複候補の除去・候補のサニタイズ・候補数の予算制限を備え、暴走・データ破壊を防止

### carapace 外部補完プロバイダ — v1.14.0

[carapace](https://github.com/carapace-sh/carapace-bin) がインストールされている場合、その膨大なコマンド定義を Tab 補完に利用できるようになりました。

- `carapace` バイナリを検出した場合に自動で有効化（`[completion] external = "auto"` 既定時）
- タイムアウト付きの外部プロセス実行基盤の上に構築（暴走時はプロセスグループ全体を kill）
- `cd` では dirs-only 防御フィルタを適用し、ファイル候補の混入を防止
- 設定のホットリロードに対応

### zsh 補完ブリッジ（常駐デーモン） — v1.14.0

zsh の補完定義（`compsys`）を jarvish から借用する zsh ブリッジを追加しました。carapace が無い環境でも、zsh の豊富な補完資産を活用できます。

- `zsh -i` を jarvish の子プロセスとして 1 本常駐させ、以後の Tab で使い回す常駐デーモン方式（既定 `external_zsh_daemon = true`）
- シェル起動直後にバックグラウンドで事前ウォームアップされるため、通常は最初の Tab の時点でウォーム状態
- `external_zsh_daemon = false` で毎回 `zsh --no-rcs` を起動するワンショット方式にも切り替え可能
- 全終了経路でデーモンを確実に shutdown し、kill/reap を非ブロッキング化

### alias 対応補完 — v1.14.0

alias 定義を展開したうえで補完するようになりました。`Shell.aliases` を `Arc<RwLock<HashMap>>` 化し、補完系と実行系で同一の alias テーブルを共有します。

## 内部実装

- **`CompletionProvider` trait 化**: 既存 3 補完（コマンド / パス / git）を trait 実装として移植し、orchestrator が複数プロバイダを連鎖評価する構成に再編（`src/cli/completer/provider.rs`）
- **外部プロバイダ実行基盤** (`src/cli/completer/external.rs`): タイムアウト + プロセスグループ kill を備えた外部補完ランナー。carapace / zsh ブリッジの共通土台
- **`RegistryProvider`** (`src/cli/completer/registry_provider.rs`, `registry.rs`): `complete` ビルトインの登録内容を Tab 補完へ橋渡し
- **zsh ブリッジ** (`src/cli/completer/zsh_bridge.rs`, `zsh_daemon.rs`): `capture.zsh` を vendor し、`zpty` 経由でワンショット / 常駐デーモンの両方式を実装。span エスケープ・symlink 対策・別プロセスグループ子孫への kill 到達を堅牢化
- **寛容スキャナと `CompletionContext`** (`src/cli/completer/context.rs`): 補完位置解析のための寛容トークナイザと文脈情報
- **ビルトインコマンド一覧テーブルの一元化** (`src/engine/builtins/mod.rs`): ビルトイン名・ヘルプを単一のテーブルに集約
- **設定** (`src/config/`): `[completion]` セクションに `external` / `external_timeout_ms` / `external_zsh_daemon` / `git_branch_commands` を追加

## 設定

`~/.config/jarvish/config.toml` の `[completion]` セクションで挙動を調整できます。

```toml
[completion]
# external = "auto"           # 外部補完の方針: "auto" | "carapace" | "zsh" | "none"
#                              # 配列 ["zsh", "carapace"] で優先順を明示指定も可能
# external_timeout_ms = 400   # 外部補完プロセスのタイムアウト（ミリ秒）
# external_zsh_daemon = true  # zsh ブリッジを常駐デーモン化するか
```

## 対応プラットフォーム

- macOS (aarch64 / x86_64)
- Linux (x86_64)

## 既知の制限

- Windows は未対応
- AI 機能の利用には OpenAI API キーが必要
- carapace 外部補完には別途 `carapace` バイナリのインストールが必要
- zsh ブリッジには `zsh` の実行環境が必要
