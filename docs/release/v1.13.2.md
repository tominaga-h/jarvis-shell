# v1.13.2 リリースノート

> 特定ディレクトリで `git status` 等を実行するとシェルが意図せず終了する致命的なバグを修正したパッチリリースです。

## バグ修正

### コマンド出力に "farewell" 等を含むパスがあるとシェルが終了する不具合を修正 — v1.13.2

通常コマンド（例: `git status`）の出力に goodbye（別れの挨拶）パターンを含むテキストがあると、それが AI の farewell 応答として誤検知され、シェルが終了してしまう不具合を修正しました。

**再現例:** 未追跡ファイルに `...corporate-farewell-...WIP.md` のような "farewell" を含むパスがあるリポジトリで `git status` を実行すると、出力の末尾が goodbye とみなされ `jarvish` が終了していました。

**原因:** goodbye 検出（farewell 文を検知してシェルを終了する機能）が、AI（Jarvis）の発話だけでなく、人間が打った通常コマンドの標準出力にまで適用されていました (`src/shell/input.rs`)。

**修正:** goodbye 検出を AI 応答経路（自然言語応答・AI パイプ）でのみ行うよう限定しました。通常コマンドの出力は、内容に farewell 文字列を含んでいてもシェル終了の判断材料にしなくなりました。判定ロジックを純粋関数 `should_exit_on_goodbye()` に切り出し、回帰テストを追加しています。

## 対応プラットフォーム

- macOS (aarch64 / x86_64)
- Linux (x86_64)

## 既知の制限

- Windows は未対応
- AI 機能の利用には OpenAI API キーが必要
