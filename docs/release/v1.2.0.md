# v1.2.0 リリースノート

## 主な機能

- **AI リダイレクト (`> ai "..."`)** ([#67](https://github.com/tominaga-h/jarvis-shell/issues/67)): コマンド出力を Jarvis に送り、対話的に分析・応答させる AI リダイレクト機能を実装（例: `git log --oneline -10 > ai "最近の変更を要約して"`）
- **機密情報のサニタイズ機能** ([#68](https://github.com/tominaga-h/jarvis-shell/issues/68)): API キーやトークン値などの機密情報が履歴や AI コンテキストに流出するのを防ぐサニタイズ機能を実装
- **AI temperature 設定** ([#66](https://github.com/tominaga-h/jarvis-shell/issues/66)): `config.toml` の `[ai]` セクションで `temperature` パラメータを設定可能に
- **CPU 使用率改善** ([#49](https://github.com/tominaga-h/jarvis-shell/issues/49)): プロンプト Git 情報取得時の CPU 使用率を大幅に改善する AsyncGitState を導入

## バグ修正

- **jarvish 内での jarvish 実行バグ** ([#71](https://github.com/tominaga-h/jarvis-shell/issues/71)): jarvish 内で jarvish を再帰的に実行できないバグを修正
- **エイリアス解除後のコマンド履歴バグ** ([#65](https://github.com/tominaga-h/jarvis-shell/issues/65)): エイリアスが解除されたコマンドが履歴に残り続けるバグを修正
- 日本語版 README のリンク切れを修正

## 内部改善

- 全体リファクタリングを実施（モジュール構造の整理）([#69](https://github.com/tominaga-h/jarvis-shell/issues/69), [#70](https://github.com/tominaga-h/jarvis-shell/issues/70))
- CPU 使用率をデバッグログに組み込み ([#56](https://github.com/tominaga-h/jarvis-shell/issues/56))
- reedline を 0.45 にアップデート
- version バッヂにリリースページへのリンクを追加 ([#64](https://github.com/tominaga-h/jarvis-shell/issues/64))
- 新しいデモ GIF を作成 ([#63](https://github.com/tominaga-h/jarvis-shell/issues/63))
- AI リダイレクトとマスキングについて README に追記
- release コマンドの手順を調整・改善

## 対応プラットフォーム

- macOS (aarch64 / x86_64)
- Linux (x86_64)

## 既知の制限

- Windows は未対応
- AI 機能の利用には OpenAI API キーが必要
