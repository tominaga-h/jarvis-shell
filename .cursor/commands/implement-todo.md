# /implement-todo コマンド

ユーザーが指定した番号のTODO項目の内容を読み取り、その機能を実際にコードベースに実装する。

## 手順

1. `docs/TODO.md` を読み込む
2. ユーザーが指定した番号 N に該当する行 `- [ ] [#N] 内容` を探す
3. 該当する項目が見つからない場合、ユーザーに「指定された番号のTODOが見つかりません」と伝える
4. 既に `- [x]` （完了済み）の場合、ユーザーに「既に完了済みです。再実装しますか？」と確認する
5. TODO項目の内容を理解し、`docs/OVERVIEW.md` を読み込んでプロジェクト全体の構成を把握する
6. 必要に応じてコードベースを調査し、関連するファイルを特定する
7. 実装計画をユーザーに提示し、確認を得る
8. 実装を行う
9. `cargo check` でコンパイルエラーがないことを確認する
10. 実装が完了したら、該当TODOの `- [ ]` を `- [x]` に変更する（`/fix-todo` と同じ処理）

## 制約

- 実装前に必ず計画を提示し、ユーザーの承認を得ること
- 既存のコードスタイル・パターンに従うこと
- 変更範囲が大きい場合は段階的に実装すること
- 実装完了後のみTODOを完了状態にすること

## 例

ユーザー入力: `/implement-todo #7`

→ `[#7] 今のJarvishだとvimとかが使えない` を読み取り、vimが使えるようにコードを修正し、完了後に `- [x]` に変更する。
